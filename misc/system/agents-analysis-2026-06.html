The Agents — Analysis & True-Agent Conversion Plan

2026-06-12 · Companion to the LifeOS Write-Up · Part 1 assesses each agent as it exists; Part 2 converts them into real agents — without the Anthropic Agent SDK and without per-token Claude billing

Part 1 — What They Are Today, Honestly

Today's "agents" are occupations, not organisms: project folders with a persona (CLAUDE.md + standing orders), domain skills, and markdown memory — animated only when a scheduled Cowork session possesses them for an hour. Between sessions they don't perceive, can't react, and hold no state beyond their files. That's not a criticism — it's a deliberately cheap design that produced months of real output. But the ceiling is visible: no event-driven behavior, no retries, no awareness between runs, and every heartbeat costs Claude tokens.

Assistant (Scheduler / Orchestrator) — Misc/

Role: the manager layer — weekly review + schedule, midweek quality gate, daily briefing, signal processing, conflict surfacing.

What works: the quality gate genuinely catches failures (W24 missing research, notify.py corruption, the 5x thesis pattern). The weekly review/schedule loop is the spine of the whole system. Coordination-file discipline is excellent.
Weaknesses: most of its job is deterministic (file checks, calendar math, overdue bumping) yet burns Claude-grade tokens; pending_actions accumulates without enforcement (April items still open); the evening reschedule engine is disabled, so overdue tasks aren't being bumped at all.

Conversion priority: FIRST. Highest mechanical share, highest frequency, clearest spec.

Finance — Finance/

Role: Monday thesis (full skill chain → DCF + comps + 1.5–2.5k words), Wednesday market scan, watchlist rotation.

What works: the strongest producer. Six theses with audited models, disciplined 12-week rotation, sector-pattern self-awareness, interview-grade output.
Weaknesses: the 5–6x identical-conclusion pattern was flagged twice and never acted on — the agent can observe its own rut but nothing forces escape; it doesn't write to Notion; the market scan is half-mechanical (data pull) and half-judgment (the take).

Conversion priority: PARTIAL. Thesis stays Claude (Tier-2 — it's the point of the system). Scan data assembly, watchlist maintenance, and earnings-calendar triggers move to the loop.

Research — Research/

Role: Tuesday queue-driven report, 2–5k words, sourced, with cross-agent recommendations.

What works: consistently synthesizes rather than summarizes; cross-agent recs (thesis-template upgrades, tutor flags) feed other lanes; queue threshold alerts work.
Weaknesses: failed silently for two consecutive weeks (ran, produced nothing, nobody knew until Thursday) — the clearest indictment of fire-and-forget sessions; queue curation depends on Eddie remembering.

Conversion priority: PARTIAL. Synthesis stays Claude. The loop owns: pre-run source gathering, output verification + retry, queue hygiene, NotebookLM mechanics.

Health — Health/

Role: Thursday midweek check, Saturday full review, Sunday meal + workout plan; adaptive periodization from Zepp + Cronometer.

What works: best persistent-memory design of any agent (baseline / metrics_history / training_log / injury_log); auto-adjust math is sound; plans respect real life (exam weeks, post-finals ramp).
Weaknesses: flying blind — HRV 4+ weeks dead, Cronometer barely logged, Notion workout logs missing — yet still burning ~3 sessions/week reviewing absent data; both CLIs broken; the agent most dependent on input discipline and least able to create it.

Conversion priority: HIGH for the data layer. Pulls, freshness checks, metrics-history appends, and the midweek check are loop work; only the Saturday review's judgment and Sunday plan composition merit Claude — gated on data freshness.

Support tracks (not agents, correctly so)

Programming (Coding/) is a PM layer over Claude Code — its judgment is the value; leave it. Tutor / Trading / Writing are skill packs invoked on demand — no schedule, no conversion needed. The bridge + dispatch worker are infrastructure, not personas; the worker's 72 Cowork polls/day is the immediate conversion target.

Part 2 — True Agents Without the SDK

The definition we're building toward

A true agent: perceives (reads its world on a trigger), decides (small model for routine, escalates for judgment), acts (tools: Notion, Calendar, files, notify), remembers (its markdown + the shared recall index), and runs without permission or per-token Claude cost.

The substrate already exists

The phone_link_bridge daemon is a Windows-resident Python process with a model router (Groq → Gemini → Ollama), DB, config, and notify plumbing. The conversion is not a new platform — it's promoting the bridge from messenger to agent host.

Architecture

agents/
  runtime.py          ← the ONE generic loop (perceive → decide → act → log)
  tools/              ← shared, whitelisted per agent
    notion.py         (Tasks, Projects, Workout, Career Hub via REST API)
    gcal.py           (read/write blocks; hard rule: never move external events)
    files.py          (domain folder R/W, dated outputs, activity_log append)
    notify.py         (existing bridge wrapper)
    recall.py         (the RAG index: recall(query) → passages)
    web.py            (fetch/search for scan data)
  personas/
    assistant.yaml    ← persona prompt distilled from standing orders,
    finance.yaml         tool whitelist, triggers, escalation rules,
    research.yaml        memory file paths
    health.yaml
  triggers.py         ← cron + events (file-watch, Gmail poll,
                         Notion dispatch queue, earnings calendar)
  escalation.py       ← writes "needs-Claude" packets to the dispatch queue

One runtime, four configs. An agent is data, not code — adding a fifth persona (Career, this fall) is a YAML file.

The decide layer — three tiers, enforced by the router

TierWhatRuns onCost
0deterministic logic: file checks, calendar math, overdue bumps, data pulls, freshness gatespure Python$0
1classification, short summaries, briefing prose, scan assembly, adherence verdictsrouter free/fast (Groq → Gemini → Ollama; local for private data)$0
2theses, research synthesis, plan composition — anything Eddie would call judgmentClaude — via escalation, never inlinesubscription

The escalation contract keeps this honest: the loop never calls Claude directly. When a persona hits Tier-2 work, it assembles a context packet (task, inputs, file paths, deadline) and writes it to the Agent Dispatch queue in Notion. A scheduled Cowork session — or Eddie, interactively — picks it up. Claude becomes a consultant the agents brief, not the engine they idle on. (When the Max plan is dry, the same packet can route to the OpenRouter/DeepSeek gateway with an explicit flag — degraded but alive.)

Health-data rule: anything personal (sleep, HRV, nutrition, injuries) routes to the local tier (Ollama) only. Free-cloud models never see health data.

Guardrails (carried over, now enforced in code)

  • Never execute trades or move money — alerts only.
  • Never move calendar events the system didn't create.
  • Notify on completion/failure; skips silent. No secrets, no raw scraped text, no stack traces to the phone.
  • Every action appends to activity_log.md — same audit trail, new author.
  • Self-verification: every artifact checked (exists, non-trivial) before a run reports success; one retry, then fail-notify. No more silent two-week failures.

Build order

  1. Weekend 1 — foundations: router keys live (Groq, Ollama, OpenRouter); dispatch poller → Python (kills the leak); bridge HTTP notify endpoint.
  2. Weekend 2 — runtime + Assistant: generic loop + notion/gcal/files tools; assistant.yaml takes over briefing, quality check, evening reschedule. Removes ~4 daily/weekly Cowork sessions alone.
  3. Week 3 — recall index: embeddings + FTS indexer over the corpus; recall() joins the toolset; Obsidian pointed at the same folders.
  4. Week 4 — Health + Finance data layers: freshness gates, pulls, scan assembly, watchlist automation; escalation packets wired for Sat review + Mon thesis.
  5. August — Research conversion + Career persona (outreach follow-up aging, coffee-chat prep packets, ND alumni pipeline) in time for move-in.

What this buys

Cost: autonomous Claude spend collapses to the Tier-2 dispatches (~thesis + research + 2 health compositions/week ≈ the ~400k tokens/week that's actually worth it). Everything else: $0, forever, regardless of what Anthropic or OpenAI charge after their IPOs.

Resilience: Max plan exhausted → the body keeps living (briefings, scheduling, health pulls, notifications, chat via local models); only the deepest thoughts pause.

Capability: event-driven behavior for the first time — earnings hit the watchlist, an email lands, a task goes overdue, and something happens within minutes instead of next Sunday.

Career capital: "I built a multi-agent OS with cost-tiered model routing, local RAG memory, and an escalation protocol to frontier models" — as a freshman. That's the AgentOS write-up's closing chapter, and it's true.

Honest caveats

Open-model tool-calling is good but not Claude-good: keep loop tasks narrow and structured (JSON in/out, schema-validated), and let anything ambiguous escalate rather than improvise. Test each persona in shadow mode (loop runs, Cowork still runs, diff the outputs for two weeks) before cutting over. The local 14B tier is the floor, not the brain — don't let Tier-1 creep into judgment work just because it's free; a wrong adherence verdict costs more than the tokens it saved.
