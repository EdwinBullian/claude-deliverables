Vibe Coder → Real AI Engineer

A field guide for going from "I prompt my way to working code" to "I actually understand what's happening under it."

You're already past the hardest part: you can ship things. The skill gap between you and a senior engineer isn't about who can type code faster — it's about fluency with the surrounding ecosystem. Knowing what to ask for. Knowing when a suggestion is wrong. Knowing the names of the parts so you can read documentation, debug confidently, and learn 10× faster.

That's what this course is. Six modules. No padding. Read in order or jump around. Each module ends with a short "what to actually do" so it doesn't just sit in your head.

How to use this Don't try to read all of this in one sitting. The vocab modules (1 and 5) are reference — skim, then come back when you hit a word you don't know. The deep ones (2, 4, 6) reward sitting with them. Ask me questions in chat whenever something doesn't land.

Reading order

If you only read three: Module 2 (Git), Module 4 (AI Engineering), Module 6 (Claude Code). Those are the highest-leverage for you specifically.

Modules 1, 3, 5 are reference material — skim them once so you know what's in them, then come back when a specific topic bites you.

What this course assumes

  • You've used Claude Code / Cowork and shipped things with it.
  • You can read code well enough to tell what it's doing at a high level.
  • You don't necessarily know why things are structured the way they are, or the names of the patterns.
  • You eventually want to build AI-native products — agents, RAG systems, AI features in tools — not just be a generalist web dev.

One more thing

Don't memorize. The names matter, but only because they let you Google effectively. The mental models matter more. If you read a section and walk away thinking "ah, that's why people do it that way" — that's the win.
